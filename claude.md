\# CLAUDE.md



\# Objetivo do Projeto



Este projeto NÃO consiste em criar um novo layout.



O objetivo é reproduzir com a maior fidelidade possível um site já existente.



O trabalho deve ser tratado como um processo de engenharia reversa da interface.



A prioridade absoluta é que o resultado visual seja praticamente idêntico ao layout de referência.



Não faça interpretações de design nem proponha melhorias estéticas.



Sempre que houver dúvida, escolha a solução que fique mais parecida com a referência.



\---



\# Filosofia de Desenvolvimento



Este projeto deve seguir o princípio:



\*\*Pixel Perfect sempre que possível.\*\*



Não tente redesenhar a interface.



Não tente modernizar.



Não tente melhorar.



Não altere proporções.



Não altere espaçamentos por preferência própria.



Não altere tamanhos porque "parecem melhores".



Copie a interface.



\---



\# Processo Obrigatório



Antes de modificar qualquer arquivo:



1\. Analise cuidadosamente a imagem de referência.



2\. Compare com a implementação atual.



3\. Liste mentalmente todas as diferenças.



4\. Corrija apenas as diferenças encontradas.



Nunca altere componentes que já estejam corretos.



\---



\# Desenvolvimento Iterativo



Nunca faça grandes alterações de uma só vez.



Após cada conjunto de alterações:



\- visualizar a página;

\- comparar novamente com a referência;

\- identificar diferenças restantes;

\- ajustar novamente.



Repita até que a interface fique praticamente igual.



\---



\# Responsividade



Todo componente deve utilizar layout fluido.



Evite valores fixos em pixels quando isso prejudicar a adaptação da interface.



Priorizar:



\- Flexbox

\- CSS Grid

\- clamp()

\- minmax()

\- max-width

\- rem

\- %



O layout deve manter a mesma aparência em diferentes larguras de tela.



\---



\# Cabeçalho



O cabeçalho deve seguir exatamente a organização da referência.



Preservar:



\- alinhamentos

\- espaçamentos

\- proporções

\- hierarquia visual



Não alterar posições sem necessidade.



\---



\# Barra de Reserva



A barra de reservas é o elemento mais importante da página.



Ela deve manter as mesmas proporções da referência.



Corrigir apenas:



\- largura

\- altura

\- alinhamentos

\- distribuição dos campos

\- margens internas



Nunca aumentar componentes apenas porque há espaço disponível.



\---



\# Tipografia



Caso necessário:



ajuste apenas:



\- font-size

\- line-height

\- font-weight



Apenas para reproduzir a aparência da referência.



\---



\# CSS



Sempre preferir:



\- Flexbox

\- Grid

\- Variáveis CSS

\- Código organizado



Evitar:



\- position:absolute desnecessário

\- margin negativa

\- valores mágicos

\- !important



\---



\# Critério de Aceitação



O trabalho somente será considerado concluído quando:



✔ A página atual e a imagem de referência apresentarem praticamente a mesma distribuição visual.



✔ As diferenças restantes forem mínimas.



✔ Todos os elementos estiverem alinhados.



✔ O layout permanecer responsivo.



\---



\# Antes de finalizar qualquer tarefa



Sempre responda:



\- O que foi alterado.

\- O que ainda está diferente da referência.

\- O que poderia ser refinado em uma próxima iteração.



Nunca considere que o layout está perfeito sem comparar novamente com a imagem de referência.
